Used Technologies

This project requires following tools and technologies:
  • Java: project was tested on version 1.8.0_221
  • Maven: project was tested on version 3.3.9
  • Spring Boot API: project was tested on version 2.3.4.RELEASE
  • GIT (optional): this tool is only required for downloading source code. Project was tested on version 2.22.0

Project Structure

This project has following structure of files:

Image: Projec structure

Step 1. Create Maven file "pom.xml"

The file pom.xml contains information about the project and configuration details used by Maven to build the project.
Important lines:
  • lines 23 - 26: Spring dependencies;
  • lines 29 - 33: removing Embedded Tomcat dependencies;
  • lines 36 - 40: enable Live reload.

Step 2. Create file application.properties

The file application.properties contains all properties used in project.
Important lines:
  • line 2: defines context path /app for this application. Please note that it works only for standalone Spring Boot. When you deploy application on Server then context path depends on package name or has to be configured on this server. The goal of adding context path property here is that URL for standalone Spring Boot or for any Server by default will be the same.

Step 3. Create class Application.java

Class Application.java enables run Sprint Boot project.
Important lines:
  • line 8: annotation @SpringBootApplication marks this class as main Application class. Attribute scanBasePackages marks packages to scan by Spring Boot;
  • line 9: class Application has to extend class SpringBootServletInitializer;
  • lines 12 - 14: overriding method configure();
  • lines 16 - 18: method main().

Step 4. Create class AbstractResponse.java

The class AbstractResponse.java contains fields which should be part of every API response.
Important lines:
  • line 7: field status contains status of response as type int. For instance status 200 means that everything is ok;
  • line 8: field message contains message of response as type String. For instance if there is some error during API call then message can contain some error details.

Step 5. Create class GreetingResponse.java

The class GreetingResponse.java contains fields specific for Greeting API call.
Important lines:
  • line 7: field text contains Greeting text as type String.

Step 6. Create class GreetingController.java

The class GreetingController.java contains all API endpoints.
Important lines:
  • line 15: annotation @GetMapping marks this method as GET endpoint. Attribute value connects this method with URL. Attribute produces indicates that result of this API endpoint will be type JSON;
  • lines 16 - 21: here GET API call is processed. Class GreetingResponse is created with OK status and text GET: Hello World!. This class is returned as a result of this GET API call;
  • line 23: annotation @PostMapping marks this method as POST endpoint. Attribute value connects this method with URL. Attribute produces indicates that result of this API endpoint will be type JSON;
  • lines 24 - 29: here POST API call is processed. Class GreetingResponse is created with OK status and text POST: Hello World!. This class is returned as a result of this POST API call.

Run Project

You can run this project:
  • Directly launching class Application.java in your IDE;
  • Or using following Maven commands:
$ mvn spring-boot:run
