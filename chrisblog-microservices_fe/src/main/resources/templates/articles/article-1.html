Used Technologies

This project requires following tools and technologies:
  • Java: project was tested on version 1.8.0_221
  • Maven: project was tested on version 3.3.9
  • Spring Boot MVC: used Back-End technology. Project was tested on version 2.3.4.RELEASE
  • Thymeleaf: used Front-End technology. Project was tested on version 3.0.4.RELEASE
  • Bootstrap: used CSS framework. Project was tested on version 5
  • GIT (optional): this tool is only required for downloading source code. Project was tested on version 2.22.0

Project Structure

This project has following structure of files:

Image: Projec structure

Step 1. Create Maven file "pom.xml"

The file pom.xml contains information about the project and configuration details used by Maven to build the project.
Important lines:
  • lines 23 - 30: Spring dependencies;
  • lines 33 - 36: Thymeleaf dependencies;
  • lines 39 - 43: removing Embedded Tomcat dependencies;
  • lines 46 - 50: enable Live reload.

Step 2. Create file application.properties

The file application.properties contains all properties used in project.
Important lines:
  • line 2: defines context path /app for this application. Please note that it works only for standalone Spring Boot. When you deploy application on Server then context path depends on package name or has to be configured on this server. The goal of adding context path property here is that URL for standalone Spring Boot or for any Server by default will be the same.

Step 3. Create class Application.java

Class Application.java enables run Sprint Boot project.
Important lines:
  • line 8: annotation @SpringBootApplication marks this class as main Application class. Attribute scanBasePackages marks packages to scan by Spring Boot;
  • line 9: class Application has to extend class SpringBootServletInitializer;
  • lines 12 - 14: overriding method configure();
  • lines 16 - 18: method main().

Step 4. Create command class GreetingCommand.java

The class GreetingCommand.java is an object whose role is to store all the information required for executing an action.
Important lines:
  • line 5: this command class stores only one field with the name text and the type String.

Step 5. Create controller class GreetingController.java

The class GreetingController.java controls the flow of the data. It controls the data flow into model object and updates the view whenever data changes.
Important lines:
  • line 11: annotation @Controller marks this class as Spring controller;
  • line 14: annotation @RequestMapping marks that this controller should be run for URL /greeting and method GET;
  • line 15: annotation @ModelAttribute connects thiw controller with command class GreetingCommand marked by name command;
  • line 17: value Hello World! is set to command field text;
  • line 18: in this line flow is redirected to view greeting/greeting.

Step 6. Create HTML file greeting.html

The file greeting.html contains all elements which are designed to be displayed in a web browser.
Important lines:
  • line 2: connection with technology thymeleaf;
  • line 5: displaying the title of page;
  • line 9: import file with custom styles from location /css/custom.css;
  • line 10: import file with custom java scripts from location /js/custom.js;
  • line 13: import file with Bootstrap styles;
  • line 14: import file with Bootstap java scripts;
  • line 17: initialize Java command class with name command;
  • line 20: displaying the value of field text from command.

Step 7. Create CSS file custom.css

The file custom.css describes the presentation of HTML elements.
Important lines:
  • lines 1 - 3: define styles for css class full-height. Element marked by this class should have height of full page.

Step 8. Create JS file custom.js

The file custom.js contains all java script methods used in project.
In this project we don't need any custom java script methods. This file should be empty.

Run Project

You can run this project:
  • Directly launching class Application.java in your IDE;
  • Or using following Maven commands:
$ mvn spring-boot:run

Demo

To see demo please copy and paste following URL to your browser's address field: http://localhost:8080/app/greeting.
